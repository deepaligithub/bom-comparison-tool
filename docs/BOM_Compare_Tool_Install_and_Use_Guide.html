Part 1 — Installation

This guide explains how to install and run the BOM Compare Tool on your machine. Windows packages are exe-based: just double-click the exe and it sets everything up; if something goes wrong (e.g. port in use), a popup message will tell you what to do.

Windows package (recommended)

If you have a package folder (e.g. BOMCompareTool-Package or BOMCompareTool-Delivery):

  • Double-click BOMCompareTool.exe — No install. The app starts and your browser opens at http://127.0.0.1:5000. If anything goes wrong, a popup explains the issue (e.g. port already in use). Data (uploads, logs) is stored in the same folder as the exe.
  • Or double-click run.bat — Installs Python dependencies (first time) and starts the app. Requires Python 3.9+ on your PATH.

Log in with admin / admin.

Prerequisites (for run.bat or source)

  • Node.js (LTS, e.g. 18 or 20) — for the React frontend. Download
  • Python 3 (3.9 or newer) — for the Flask backend. Download
  • npm — included with Node.js

Windows: Add Python to your PATH (e.g. C:\...\Python3xx\ and C:\...\Python3xx\Scripts\).

Option A: Local development (recommended for first-time setup)

1. Get the project
Clone or download the project and open a terminal in the project root (the folder that contains backend and frontend).

2. Install and run the backend

cd backend
pip install -r requirements.txt
python run.py

Backend runs at http://localhost:5000. Leave this terminal open.

Optional — virtual environment (recommended):

cd backend
python -m venv venv
venv\Scripts\activate    # Windows
pip install -r requirements.txt
python run.py

3. Install and run the frontend
Open a second terminal in the project root:

cd frontend
npm install
npm start

The app opens at http://localhost:3000. If port 3000 is in use, use the port suggested (e.g. 3001).

4. Verify
Log in with admin / admin. Backend health: http://localhost:5000/api/health should return "status": "ok".

Option B: Production run (single server)

Build the frontend once, then run the backend to serve the app:

cd frontend
npm install
npm run build

Then run the backend with the production server (e.g. Waitress):

cd backend
pip install -r requirements.txt
python run_production.py

API runs at http://127.0.0.1:5000. Serve the built frontend from the same host (e.g. copy frontend/build into backend/app/static) or configure your reverse proxy.

Option C: Windows desktop app (Electron)

From the project root in PowerShell:

.\scripts\build-windows-app.ps1

Prerequisites: Node.js, Python 3, and pip install pyinstaller waitress. The script builds the frontend, packages the backend, and produces installers and MSIX in electron/dist-electron/.

Where data is stored

  • Users and auth: backend/uploads/auth_users.json. Default user: admin / admin (role: admin, plan: paid).
  • Saved mappings: Under backend/uploads/.
  • Logs: backend/logs/ (if configured).

Port 5000 already in useStop the other process or set BOM_PORT (e.g. 5001).

Part 2 — How to Use

This section walks you through logging in, comparing two BOMs, viewing results, and using optional features such as Mapping Manager and export.

1. Log in

  1. Open the app in your browser. The app usually opens automatically when you start it; if not, use the address shown when you started the app.
  2. Enter Username: admin, Password: admin.
  3. Click Log in.

2. Compare two BOMs (main workflow)

2.1 Upload Source BOM and Target BOM

  • Source BOM = reference file (e.g. from your PLM or source system).
  • Target BOM = file to compare (e.g. from SAP or target system).

On the BOM Compare page: use Upload Source BOM and Upload Target BOM to select files. Supported formats:

FormatSource BOMTarget BOM
CSVYesYes
Excel (.xlsx)YesYes
JSONYesYes
PLMXMLYesNo

2.2 Optional: Use a saved mapping preset
If an admin has set an active mapping in Mapping Manager, check “Use saved mapping preset” before comparing. Otherwise the tool builds a mapping from file headers (column names, case-insensitive).

2.3 Run the comparison
Click Validate BOMs (or press Ctrl+Enter). The page scrolls to the results when done.

2.4 Understand the results

  • Summary — Total rows and counts for Matched, Different, Source BOM only, Target BOM only.
  • Column mapping — Which columns were compared and which exist only in one file.
  • Results table — One row per key; columns show Source/Target values and Status (Matched, Different, Source BOM only, Target BOM only).

Filters: All, Matched, Different, Source BOM only, Target BOM only. Use the search box to filter by any visible column. Use Columns ▾ to show or hide columns. Click the ℹ button on a row for full detail.

3. Export results (paid plan)

If your account has Export: Export CSV and Export PDF in the toolbar above the results table.

4. Ignored rows and columns

Use the Ignored area to inspect duplicates, missing keys, and skipped columns. You can export ignored rows as CSV from that section.

5. Mapping Manager (paid plan or admin)

Create, edit, and activate saved mappings (presets) for files with different column names or for PLMXML.

  • Open Mapping Manager from the navbar.
  • Create Mapping — Use UI (upload samples, map columns, set keys) or Manual (paste mapping lines).
  • Manage Mappings — Set one mapping as Active. On BOM Compare, check “Use saved mapping preset” and run Validate BOMs.

6. Change password

Click your username in the top-right → Change password → enter current and new password.

7. Users page (admin only)

Admins see a Users link in the navbar to view users and change plans (free/paid).

8. Sample files

Use files in the samples/ folder: bom_a_* as BOM A and bom_b_* as BOM B (same number for a pair). Includes 1000-row samples for large-BOM testing; the app supports up to ~10,000 lines per BOM (within the 10 MB upload limit).
